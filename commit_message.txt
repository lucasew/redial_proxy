🧹 Janitor: Fix timer leak and slog error handling in dialer

Addresses technical debt in `internal/dialer/dialer.go`:
1. Replaced `time.After` with `time.NewTimer` to prevent goroutine/timer leaks when contexts are cancelled early, stopping the timer explicitly.
2. Updated `slog.Warn` to pass the `err` object directly instead of calling `.Error()`, leveraging structured logging capabilities.

Assumptions:
- The context cancellation handling is a critical path for network operations and optimizing it improves reliability.

Alternatives Not Chosen:
- Implementing custom timeout handling: Using standard library context features combined with `time.NewTimer` is cleaner and standard practice in Go.

How To Pivot:
- If `time.After` is preferred, we could document the known minor memory leak, but it's universally recommended against in `select` with contexts.
- If older Go versions are required where timer behavior differs, a different backoff mechanism might be needed.
